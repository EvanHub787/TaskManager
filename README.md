# TaskManager

Issue 対応フロー、Todo 受信箱、メンバー、案件進捗をローカルで管理する軽量ツールです。

## 機能

- Issue 看板: デフォルトフローは `調査中 -> 修正中 -> テスト中 -> MR`
- フロー編集: Issue ステップの追加、名称変更、削除
- Todo: 手動並べ替え、Issue との関連付け、日次整理、完了日の管理
- 日報: 今日の完了、対応中、次の予定、期限超過・停滞をテキストで生成
- 検索: 通常検索に加え、`担当:自分`、`案件:案件名`、`期限:今日` などの条件検索
- 状態管理: 操作の取り消し、操作履歴、自動復元ポイント、Issue の停滞表示
- 案件管理: 案件別の集計と詳細表示
- メンバー管理: メンバーの追加、名称変更、削除
- データ保存: IndexedDB、JSON インポート/エクスポート、データファイル保存

## 使い方

ブラウザで `index.html` を直接開きます。

ローカルサーバーで開く場合:

```bash
python3 -m http.server 4173
```

その後、次の URL を開きます。

```text
http://localhost:4173/
```

## データについて

デフォルトではブラウザの `IndexedDB` に保存されます。既存データとの互換性維持とフォールバックのため、次の `localStorage` key も継続利用します。

```text
follow-manager-v1
```

定期的に右上のエクスポート機能で JSON バックアップを作成してください。File System Access API 対応ブラウザでは「データファイルを開く / データファイルへ保存」も利用できます。
